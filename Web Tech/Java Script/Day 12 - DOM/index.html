Document Object Model

  • The Document Object Model (DOM) is a programming interface from web documents that represents the html or XML documents as a tree structure, where each node represents an element, attribute, or piece of text in th document.
  • When a web page is loaded, the browser creates a dom tree that represents a document's structure and content
  • Each node in the tree is represented as an JS object, which we can access and manipulate using the DOM API.

What is DOM API?

  • The DOM API(Application Programming Interface) is a set of programming interfaces and method that allow developers to interact with the DOM tree and manipulate the content and structure of the web document.
  • The DOM API provides a standardized way to create, modify, and delete elements and attributes, change style and classes, handle events and more.

Methods:

  • getElementById('id_name'): It returns reference of single elements object where id name matches.
  • getElementByClassName('class_name'): It return html collection all elements matches with class name.
  • getElementByTagName('tag_name'): it return html collection all elements matches with tag name.
  • querySelector('css_selector'): It return reference of the first element that matches a specified CSS selector.
  • querySelectorAll('css_selector'): It return Node list of all elements that matches a specified css selector.

Difference between HTMLCollection and NodeList.

HTMLCollection

  • Live updates automatically when DOM changes.
  • Only contains element nodes (no text or comment)
  • Accessed by name, id or index
  • Returns only element (e.g., div , p)

NodeList

  • Usually static (But some are live like childNodes)
  • Can contain any node type
  • Accessed only by index
  • Nay include text, nodes, comments
