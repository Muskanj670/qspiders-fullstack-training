GEC - Global Execution Context

When we give js code to the browser, JS engine will allocate(create) a global memory block to the execution of js code

Here, we have a window variable which have referenced to GEC.

JS code run in two phases

  1. Variable Phase / Memory creation phase / Compilation phase
    • In variable phase, js engine will check the complete js code and it will search for variable declaration statement.
    • if variable is declared them js engine allocate memory for them.
    • variable declared with var will be initialized storing "Undefined" at the time of memory block creation. Variable declared with let and const will remain uninitialized at the time of memory block creation
  2. Execution phase:
    • In this, JS engine will execute the instructions line by line
