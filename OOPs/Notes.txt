OOPs: Object Oriented Programming System
In python, if we want to work with operations we use programming but anything related to real time senarious we use OOPs.
Or, it is nothing but relationship btw class and objects.

Class: It is a blueprint for the operation.
Object: It is an instance of the class.
Syntax: 

class class_name:
    Properties
obj = class_name()

Example:
class abc:
    pass
obj = abc()
print(type(obj))
-----> <class:'__main__.abc'>

**While assigning properties to any class or object we use keys and values.

In OOPs properties of class is independent whereas object in dependent.

#Members / properties of class and objects

        -> These are the properties which belongs to class AND objects.
        -> We have two types of Members :
            a. Static - These are the properties belongs to class
            b. Object Members - These are the properties belongs to objects


# Methods: Functions inside the class.

# Constructor / __init__():
    - It is a special method which is used to initialize the members of the object.
    - It is used to define the object properties.
    - self is the first and mandatory argument inside the Constructor.
    - self is used to carry the address of the object.
    - We don't need to call constructor outside the class, it is invoked at the time of object creation.

    --> Syntax:
            class class_name:
                def __init__(self, arg1, arg2, ....):
                    self.arg1 = arg1
                    self.arg2 = arg2
            obj1 = class_name(val1, val2, ....)
