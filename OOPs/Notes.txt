OOPs: Object Oriented Programming System
In python, if we want to work with operations we use programming but anything related to real time senarious we use OOPs.
Or, it is nothing but relationship btw class and objects.

Class: It is a blueprint for the operation.
Object: It is an instance of the class.
Syntax: 

class class_name:
    Properties
obj = class_name()

Example:
class abc:
    pass
obj = abc()
print(type(obj))
-----> <class:'__main__.abc'>

In OOPs properties of class is independent whereas object in dependent.

# Methods: Functions inside the class.

# Constructor / __init__():
    - It is a special method which is used to initialize the members of the object.
    - It is used to define the object properties.
    - self is the first and mandatory argument inside the Constructor.
    - self is used to carry the address of the object.
    - We don't need to call constructor outside the class, it is invoked at the time of object creation.

    --> Syntax:
            class class_name:
                def __init__(self, arg1, arg2, ....):
                    self.arg1 = arg1
                    self.arg2 = arg2
            obj1 = class_name(val1, val2, ....)

# Methods in class:
    - These are the functions inside the class.
    - Their are 3 types of methods inside the class:
        --> Class method
        --> Object / instance method
        --> Static Methods

    --> Class Methods: 
        - It is used to access and modify the members of the class.
        - '@classmethod' decorator is used to define class method.
        - 'cls' is the first and mandatory argument in the decorator, which holds the address of the class.
        - To call the classmethod class_name is used.

        --> Syntax:
                class class_name:
                    @classmethod
                    def method_name(cls, args):
                        method body
                class_name.method_name(args)

    --> Instance Methods:
        - It is used to access and modify the members of the object.
        - It takes the first and mandatory argument 'self' inside it.

        --> Syntax:
                class class_name:
                    def method_name(self):
                        SB

                obj = class_name(args)
                obj.method_name()

    --> Static Methods:
        - It is used to store extra functionalities.
        - Used as a utility or helper method.
        - '@staticmethod' decorator is used to define static method.
        - It does not take any mandatory argument.
        - We can call this method by using both class as well as object.

        --> Syntax:
            class class_name:
                @staticmethod
                def method_name(args):
                    SB
                
            obj = class_name()
            obj.method_name(args)
            class_name.method_name(args)
